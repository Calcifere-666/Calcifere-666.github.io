On veut le chiffre d’affaires généré par catégorie desserts ou catégorie :
Cette requête est importante si Kdou veut savoir combien génèrent-t-il et les aider à investir ou réduire selon leur politique. Code SQL :

SELECT SUM(D.PrixUnitCli * D.QteCom) AS chiffreaffaire, C.NomCateg
FROM categorie AS C, fournisseur AS F, produit AS P, detailcommande AS D
WHERE P.NoFour = F.NoFour
AND D.RefProd = P.RefProd
AND C.CodeCateg = P.CodeCateg
AND C.NomCateg LIKE "Desserts"
GROUP BY C.NomCateg;

UNION ALL SELECT SUM(D.PrixUnitCli * D.QteCom) AS chiffreaffaire, C.NomCateg
FROM categorie AS C, fournisseur AS F, produit AS P, detailcommande AS D
WHERE P.NoFour = F.NoFour
AND D.RefProd = P.RefProd
AND C.CodeCateg = P.CodeCateg
AND C.NomCateg LIKE "Boissons"
GROUP BY C.NomCateg;

Proposition d'indicateur

Nous avons eu pour mission aussi de proposer des indicateurs comme ci-dessus afin de faire évoluer Kdou :
Cette requête aide KDou à identifier les clients les plus éloignés, permettant ainsi de planifier des livraisons optimisées. En regroupant ou en espaçant les livraisons pour ces clients, KDou peut réduire les trajets fréquents, diminuant les coûts logistiques et l'empreinte carbone liée au transport, contribuant ainsi à ses engagements environnementaux et économiques. Code SQL :

SELECT Client.SocieteCli, AVG(Commande.DistanceLivraison) AS DistanceMoyenne
FROM Client INNER JOIN Commande ON Client.CodeCli = Commande.CodeCli
GROUP BY Client.SocieteCli;

Cet indicateur sert à identifier les produits dont la quantité en stock (QteParUnite) est inférieure à un certain seuil (p.e, 10 unités). Pour pouvoir anticiper des ruptures de stock et de passer des commandes auprès des fournisseurs concernés. Code SQL :

SELECT Produit.NomProd, Produit.QteParUnite, Fournisseur.Societe, Fournisseur.Pays
FROM Produit P
JOIN Fournisseur F ON P.NoFour = F.NoFour
WHERE P.QteParUnite < 10;

Cet indicateur indique les produits et leur nombre de quantité vendus. Avec ce chiffre, on pourrait mettre en valeur certains produits et agrandir leurs stocks, ou au contraire réduire ceux qui n'ont pas beaucoup de commandes afin d'avoir un maximum de rendement au lieu d'avoir du stock pour rien et perdre de l'argent. Code SQL :

SELECT Fournisseur.Pays, Count([2c Sr].RefProd) AS NBProdVendu, Sum(Produit.UniteStock) AS NbStockProd
FROM Fournisseur INNER JOIN ([2c Sr] INNER JOIN Produit ON [2c Sr].RefProd =
Produit.RefProd) ON Fournisseur.NoFour = Produit.NoFour
GROUP BY Fournisseur.Pays
ORDER BY Count([2c Sr].RefProd) DESC;
